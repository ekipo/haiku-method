Spec notes for this wireframe:
  • All colors use var(--color-upload-affordance-*) and var(--color-feedback-pending-*) tokens — no raw hex, no raw Tailwind palette classes.
  • Drop zone carries role="button" and aria-label="Upload knowledge file" exactly as specified.
  • Uploading state sets aria-busy="true" on the button; adjacent aria-live="polite" region announces "Uploading {filename}".
  • Uploaded state announces "Uploaded {filename}" via the same live region.
  • Error state announces error text via the live region.
  • Empty state shows visible text "drop files or click to browse" — not a placeholder image.
  • Mobile: drop zone collapses to a single <button> that opens native file picker; no drag-drop on touch. All controls ≥44px.
  • Touch targets: .touch-target--hit-area applied to × remove buttons (sub-44px visual).
  • Reduced-motion: drag-over scale animation suppressed; progress bar remains.
  • Direction A passive-observer constraint: zero classification-trigger controls on this surface (per SPA-UI-SPECS.md §0).
