Rollback reason banner · blocked user-gate state

Two coupled surfaces. The banner fires when the FSM rolls a stage back to elaborate. The blocked state renders when the reviewer opens the UI before the gate is reachable.

unit-08 · FB-07

Surface decisions

  • Rollback banner is a full-width top-of-content banner (below header, above the main review surface). It fires once per rollback event and persists until the reviewer dismisses it or the stage re-enters review. Not a modal — modals steal focus and the reviewer may need to inspect artifacts before acting.
  • Blocked user-gate state replaces the user-gate action bar (the bottom-of-sidebar Approve / Request Changes bar) with a non-interactive explanatory panel. It's contextual — only shows while feedback is pending.
  • The two surfaces never render at the same time: rollback banner = "stage just rolled back, here's why"; blocked gate = "stage hasn't hit the gate yet, here's the blockers".

Interactive states · accessibility · responsive

States (banner)

  • default — amber (still-pending / write-failure) or rose (assessor-error); persistent, non-modal.
  • hover — dismiss button hover:text-amber-300 / hover:text-rose-300.
  • focus — focus-visible ring on dismiss, "view log" — focus:ring-2 focus:ring-amber-500.
  • dismissed — collapses to a single-line strip pinned above content, click expands.
  • persistence — banner session-scoped; stored in sessionStorage keyed by {stage}:{visits}, re-shows on new rollback.

States (blocked gate)

  • default — non-interactive; list of blockers with badge + id + title.
  • hover — blocker row hover:bg-amber-900/20 / hover:bg-amber-100; cursor pointer — clicking scrolls to item in feedback list.
  • focus — each blocker row is a button for keyboard nav, focus ring visible.
  • disabled — Approve and Request Changes buttons are disabled with aria-disabled="true", cursor-not-allowed, muted styling. Must use aria-describedby pointing to the blocker copy so screen readers read the reason.
  • transition — when assessor succeeds, this whole panel dissolves (0.3s opacity) and the normal action bar fades in.

Accessibility + responsive

  • Banner = role="status" aria-live="assertive" (rollback is a state change the reviewer needs to hear).
  • Blocked gate = role="region" aria-label="Approve gate blocked".
  • Contrast: amber-200/amber-900/15 > 7:1 (dark), amber-900/amber-50 > 7:1 (light).
  • Icons are aria-hidden="true"; meaning comes from pill text and heading.
  • Touch targets: dismiss + "view log" buttons are 28×28 visible, 44×44 hit area via ::before extension.
  • mobile 375px — banner stacks badge + visit counter below headline; action buttons remain 2-col.
  • tablet 768px — banner at full width of main content; sidebar bottom sheet hosts blocked-gate panel.
  • desktop 1280px (xl) — banner centered within max-w-page; blocked-gate panel fits within the w-80 xl:w-96 sidebar (widens to 384px at xl).
